# Tech Stack

## Framework and Runtime

- **Application Framework:** Astro
- **Content Format:** Markdown / MDX
- **Architecture:** Static-first, content-driven
- **Package Manager:** npm (default with Astro)

## Frontend

- **JavaScript Framework:** Astro (islands architecture; minimal JS by default)
- **CSS Approach:** Custom CSS, typography-focused
- **UI Components:** Minimal, custom components only when needed

## Content Management

- **Content Source:** Local Markdown/MDX files in repository
- **Version Control:** Git via GitHub (public repository)
- **CMS:** None (explicit non-goal)

## Deployment and Infrastructure

- **Hosting:** Vercel
- **Domain:** sarinsaurabh.in
- **CI/CD:** Vercel automatic deployments from GitHub

## Explicit Non-Goals

The following are intentionally excluded from the current scope:

- **CMS:** No external content management system; content lives in the repo
- **Analytics:** No analytics-heavy setup; focus on content, not metrics
- **Complex Interactivity:** Static-first; JavaScript only where essential
- **SEO Optimization:** Not optimizing for search; optimizing for signal
- **Monetization:** No ads, sponsorships, or paid features

## Design Principles

- **Typography-focused:** Visual design serves readability and calm tone
- **Minimal dependencies:** Avoid heavy frameworks; keep build simple
- **Content-first:** Architecture optimized for writing and updating Markdown
- **Fast and simple:** Static output; no runtime complexity
